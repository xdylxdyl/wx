NOTICE: it's conflict with spring bean name autoproxy. so DON"T USE THEM together.

SEE spring reference chapter 6:

[Warning] Warning The <aop:config> style of configuration makes heavy use of Spring's auto-proxying mechanism. This can cause issues (such as advice not being woven) if you are already using explicit auto-proxying via the use of BeanNameAutoProxyCreator or suchlike. The recommended usage pattern is to use either just the <aop:config> style, or just the AutoProxyCreator style.
